# ДЗ № 1. Примеры использования ClickHouse в реальных проектах

Этот документ содержит подборку кейсов внедрения ClickHouse в компаниях. Для каждого кейса указаны: компания, специфика проекта, особенности внедрения и архитектура.

## Пример 1: Яндекс Метрика (Яндекс)

### Специфика проекта

ClickHouse стал основным хранилищем и движком для аналитических запросов в системе веб‑аналитики. Ежедневно обрабатываются миллиарды событий: просмотры страниц, клики, конверсии, сессии.

### Особенности внедрения и архитектура

- Переход на ClickHouse состоялся ещё в 2014 году.
- Для высокой пропускной способности при вставке данных использовали **буферные таблицы**: данные сначала попадают в буфер, а потом пачками сливаются в основную таблицу.
- Применяли партиционирование и индексы пропуска для ускорения сложных агрегаций.
- Реализовали репликацию для отказоустойчивости и шардирование для горизонтального масштабирования.
- Данные поступают через кастомные коннекторы и внутренние сервисы Яндекса; часть данных предварительно агрегируется до вставки в ClickHouse.
- В архитектуре используются движки таблиц `MergeTree` и `AggregatingMergeTree`, а также проекции и материализованные представления для оптимизации частых запросов.

### Ресурсы

## Пример 2: Cloudflare (веб‑безопасность и CDN)

### Специфика проекта

Cloudflare использует ClickHouse для хранения и анализа логов запросов к своим серверам: HTTP‑ и DNS‑трафика, логов, управления ботами, анализа работы Cloudflare Worker. Это позволяет в реальном времени строить дашборды (например, Firewall Analytics, Cloudflare Radar), выявлять аномалии, DDoS‑атаки и формировать отчёты по безопасности.

### Особенности внедрения и архитектура

- Cloudflare — один из ранних адаптеров: запустили в продакшене ещё в конце 2016 года.
- Масштаб: более тысячи активных реплик, обработка сотен миллионов вставок строк в секунду.
- Активно применяли **словари (Dictionaries)** для эффективных lookup‑операций (чтобы не делать дорогие JOIN).
- Столкнулись с проблемой потребления памяти: крупнейший словарь занимал более 20 ГБ, а общий объём всех словарей превысил 15 ТБ. Решение — переход с хэшированной схемы на «хэшированный массив» (hashed array), что сократило потребление памяти более чем в 4 раза.
- Разработали инструмент **ClickHouse Dictionary Dumper**, чтобы отделить производство данных от их потребления и устойчиво обрабатывать сбои в upstream‑системах.
- Архитектура: сбор логов с пограничных серверов → парсинг и нормализация → пакетная вставка в ClickHouse → хранение на `MergeTree` с партиционированием по времени и TTL для автоматического удаления старых данных → аналитика через Grafana и внутренние инструменты.

### Ресурсы

- [Статья в блоге ClickHouse «How Cloudflare Processes Hundreds of Millions of Rows per Second with ClickHouse»](https://clickhouse.com/blog/how-cloudflare-processes-hundreds-of-millions-of-rows-per-second-with-clickhouse) — детальный разбор от инженера Cloudflare (James Morrison).
- [Репозиторий `clickhouse-presentations` на GitHub](https://github.com/ClickHouse/clickhouse-presentations) — в папке `2018-meetup14` лежит PDF‑презентация с митапа, где Cloudflare делилась опытом.

## Пример 3: ВКонтакте (продуктовая аналитика)

### Специфика проекта

ClickHouse использовали для продуктовой аналитики: агрегации событий пользователей, расчёта метрик удержания, частотности взаимодействий, соединения агрегатов по разным продуктам.

### Особенности внедрения и архитектура

- Для вставки данных с десятков тысяч серверов применяли локальные агенты («Киттенхаус»), которые буферизировали события и периодически сбрасывали их в ClickHouse.
- Использовали движок `ReplicatedReplacingMergeTree` с тонкой настройкой партиционирования и сэмплирования (`SAMPLE BY`) для балансировки между точностью и производительностью.
- Применяли внешние словари для ускорения запросов.
- В ETL/ELT‑процессах реализована обработка данных в потоковом и пакетном режимах, обогащение справочниками и дедупликация.
- Доступ к данным — через внутренние BI‑инструменты, API для построения отчётов и интеграцию с инструментами визуализации.

### Ресурсы

- [Статья на Хабре «Как VK вставляет данные в ClickHouse с десятков тысяч серверов»](https://habr.com/ru/companies/ua-hosting/articles/483712/) — подробный разбор архитектуры, проблем при высокой нагрузке и способов их решения.
